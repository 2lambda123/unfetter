        Intrusion Set Dashboard - Contains a graphical representation of the different

into the system.

        Partners - Information about Unfetter’s partners.

Assessments:

The assessments page contains three separate questionnaires (Indicators, Mitigation, and Sensors) about a network. To start a new assessment, click on one of the corresponding buttons. Each assessment has pre-defined answers via a dropdown arrow.  Once an assessment is completed and saved, it will show up in the list on this assessment page. Each completed assessment can then be viewed (summary of results), edited, or deleted.

Summary Page:  The summary for each of the assessments can be viewed by selecting the red hand icon next to the assessment.  

The top part of the summary page displays the overall risk level and the areas that are the most vulnerable.  

The bottom of the page has a couple graphs that shows the percentage of coverage versus one of the pre-defined assessment values (which can be modified via a dropdown above the graphs).  The exception to this is the Sophistication bar graph, which displays the risk level based on the attacker’s skill level to execute the techniques.

Novice - Demonstrates a nascent capability. A novice has basic computer skills and likely requires the assistance of a Practitioner or higher to engage in hacking activity. He uses existing and frequently well known and easy-to-find techniques and programs or scripts to search for and exploit weaknesses in other computers on the Internet and lacks the ability to conduct his own reconnaissance and targeting research.

Practitioner -  Has a demonstrated, albeit low, capability. A practitioner possesses low sophistication capability. He does not have the ability to identify or exploit known vulnerabilities without the use of automated tools. He is proficient in the basic uses of publicly available hacking tools, but is unable to write or alter such programs on his own.

Expert - Demonstrates advanced capability. An actor possessing expert capability has the ability to modify existing programs or codes but does not have the capability to script sophisticated programs from scratch. The expert has a working knowledge of networks, operating systems, and possibly even defensive techniques and will typically exhibit some operational security.

Innovator -         Demonstrates sophisticated capability. An innovator has the ability to create and script unique programs and codes targeting virtually any form of technology. At this level, this actor has a deep knowledge of networks, operating systems, programming languages, firmware, and infrastructure topologies and will demonstrate operational security when conducting his activities. Innovators are largely responsible for the discovery of 0-day vulnerabilities and the development of new attack techniques.

For more detailed information, the assessment report can be viewed by clicking on the assessment name (blue link in below image) in the list of assessments.

The assessment report will display the 10 kill chains (tactics) down the left side and risks percentages on the other half of the page.  

Each of the kill chains on the left can be used to drill-down further in the report by clicking on them.  Each kill chain report will then highlight the attack patterns for that kill chain.  By selecting an attack pattern, a summary of that attack will then be displayed on the right of the page.

Intrusion Set Dashboard:  

The intrusion set dashboard lists all of the intrusion sets that are in the system and the associated attack patterns that each uses as well as the Critical Security Controls (CSCs) for each.  An intrusion set can be selected from the left hand side of the page and then the corresponding data will be highlighted on table and graphs.

STIX Object UI:  The STIX Object UI is another locally hosted UI (https://localhost/cti-stix-ui/) that controls the input of data into the system using the STIX standard language (STIX 2.0).  

The STIX objects allows for data to be entered for the following data types via the STIX Object dropdown:

 

Attack Patterns - Each attack pattern is a type of TTP that describes behaviors and actions that adversaries may take in your network.  

Campaigns - A grouping of adversarial behaviors that describe a set of malicious activities or attacks that occur over a period of time against a specific set of targets.  Campaigns usually have well defined objectives and may be part of an Intrusion Set.

Course of Actions - Action taken to prevent and attack or respond to an attack that is in progress.  It could be described as a Critical Control or Mitigation.  It could be technical, automatable responses or analytical, but it could also represent higher level actions like employee training or penetration testing.  For example, a Course of Action to apply security patches could prevent vulnerability exploitation.

Indicators - Contain a pattern that can be used to detect suspicious or malicious cyber activity.

Identities - Represent actual individuals, organizations, or groups as well as classes of individuals, organizations or groups.

Malware - Relationships explicitly defined between the Malware object and other objects.  The first section lists the embedded relationships by property name along with the corresponding target.  The rest of the table identifies the relationships that can be made from the Malware object by way of the Relationship object.  The reverse relationships(relationships to the Malware object) are included as a convenience. 

Sightings - A time in which a particular Campaign, Intrusion Set, or Incident was observed.

Tools - Tools are legitimate software that can be used by threat actors to perform attacks.Unlike malware, these tools or software packages are often found on a system and have legitimate purposes for power user and administrators.  Remote access tools such as RDP and networking scanning tools such as NMap are some examples.

Threat Actors - Individuals, groups, or organizations believed to be operating with malicious intent.  Threat actors can be characterized by their motives, capabilities, intentions, sophistication level, past activities, resources they have access to and their role in the organization.

Intrusion Sets - Grouped set of adversarial behaviors and resources with common properties that is believed to be orchestrated by a single organization.
