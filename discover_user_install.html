Installation

The entire application, Unfetter Discover, is made up of multiple docker containers. Each builds upon the other. We use Docker Compose to manage the startup and shutdown of all the Docker containers.

The Github organization unfetter-discover houses a GitHub project for every container.

The most common way to use Unfetter-Discover is to install it using the pre-compiled containers from Docker Hub.
Mac User
First, you will need to clone the unfetter project from Github. Then, run docker-compose with the default arguments. This will pull in all the docker containers from Docker Hub and bring up the application.

Step 1: Create a directory where you want the project to be built and navigate to it.

Step 2: Clone the unfetter project from github.

git clone https://github.com/unfetter-discover/unfetter.git

Step 3: navigate into the unfetter directory.

cd unfetter

Step 4: Bring up all the containers using docker-compose

docker-compose up

After docker-compose up finishes running, the application should be up and running.

Open up a web-browser(chrome/firefox) and navigate to https://localhost/unfetter-discover-ui/

Note: if you receive a 404 error from nginx, ensure you include the trailing slash on the URL

Unfetter-Discover will create certs and store them locally. You will need to accept the certificates to move forward.
Windows (Using Docker Toolbox):
Step 1: First, you will need to bring up Docker and the VirtualBox environment. Double-Click on the Docker Quickstart Terminal Icon on the desktop. (First time through, a few windows may be minimized in the taskbar that will ask for permission to configure network.)

Step 2: After the environment is setup, you should get a $ prompt. Before continuing, lets double check the IP address of the docker-machine running (we'll need this later to get to the application). At the prompt, run "docker-machine.exe ip" This will tell us the IP of the docker machine (usually 192.168.99.100)

docker-machine.exe ip

If this command doesn't return an IP address, run the following command and try again.

docker-machine.exe env

docker-machine.exe ip

Step 3: Now, create a directory to put the unfetter project into and navigate to it.

mkdir unfetter-discover

cd unfetter-discover

Step 4: Once we're in the directory, we can now clone the unfetter project from github.

git.exe clone https://github.com/unfetter-discover/unfetter.git

Step 5: navigate into the unfetter directory

cd unfetter

Step 6: Bring up all the containers using docker-compose. Since we are using VirtualBox to run docker, we have to specify a specifig yml script.

docker-compose.exe -f docker-compose.virtualbox.yml up

After docker-compose up finishes running, the application should be up and running.

(command prompt should display"unfetter-discover-ui | Build successful -ms.")

Open up a web-browser(chrome/firefox) and navigate to https://'dockermachine ip'/unfetter-discover-ui/ The docker machine IP is the IP we found in Step 2 above but is usually 192.168.99.100

https://192.168.99.100/unfetter-discover-ui/

Unfetter-Discover will create certs and store them locally. You will need to accept the certificates to move forward.

Important:Once finished with the application, a Ctrl-C in the command prompt window will shut-down all the docker containers and return to the $ prompt. Next, you will need to open the VirtualBox application by double clicking the Oracle VM VirtualBox shortcut on the desktop. Once there, you can either right click on the "default" machine (should be running) and select close->save state or get to it from the machine dropdown in the menu. This is an important step because if you do not save the state of the VirtualBox VM, any data imported into the application will be lost.

Alternately, you could ctrl-c in the command prompt window to shut-down the docker containers and then run the following command:

/c/'Program Files'/Oracle/VirtualBox/VBoxManage.exe controlvm default savestate

Leveraging The Community

Unfetter makes it easier to get started and take advantage of community threat models like MITRE's ATT&CK™. Cyber security professionals can use Unfetter to establish a baseline security posture, explore relationships to identify gaps, and experiment with defensive courses of action before the threat hits. You can read more about MITRE's efforts at the MITRE GitHub Page.

security

Analytics in Context

Our initial prototype, Unfetter Analytic, is a reference implementation of a platform designed to help analytic developers experiment and gain familiarity with the ATT&CK™ framework as a means of measuring the effectiveness of analytics.

settings

Discover Gaps

Our second prototype, Unfetter Discover is aimed at helping the user explore additional gaps across their security posture and experiment with next steps. The initial focus is on mapping mitigations and security controls to ATT&CK™ and demonstrating how to explore, learn, and communicate between tactical, operational, and strategic levels of operation.
